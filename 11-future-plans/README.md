# 🚀 Future Plans

## 📌 Overview

This section outlines the planned evolution of the homelab, focusing on improving system architecture, reliability, and separation of roles.

---

## 🧠 Infrastructure Direction

The lab is evolving into a structured environment with clearly defined roles:

- 🥧 Raspberry Pi → Network services (DNS, Pi-hole, VPN)
- 🧠 Mini PC → Dedicated server / NAS
- 🎮 Secondary PC → Guest workstation
- 🎮 Main PC → Personal use and gaming

---

## 🥧 Raspberry Pi Deployment

### Goals:
- Deploy Pi-hole for DNS filtering
- Provide network-level ad blocking
- Add optional VPN (WireGuard)

### Future:
- Integrate with monitoring tools
- Improve network segmentation

---

## 🧠 Mini PC Server / NAS

### Goals:
- Run Docker containers
- Host media server (Jellyfin)
- Provide file storage (NAS)
- Run game servers (e.g., Minecraft)

### Migration Plan:
- Move server workloads from secondary PC
- Optimize storage and services
- Ensure stable, always-on operation

---

## 🎮 Secondary PC (Guest Workstation)

### Goals:
- Maintain as a clean and ready-to-use system
- Support gaming and general usage for guests
- Use for testing environments when needed

---

## ⚡ Power Infrastructure

### Goals:
- Maintain safe and stable power distribution
- Avoid overload and unsafe configurations

### Future Improvements:
- Add UPS for critical systems
- Improve cable management
- Ensure reliability for always-on devices

---

## 🌐 Network Improvements

- Introduce better network structure
- Consider VLANs in the future
- Improve monitoring and visibility

---

## 📈 Long-Term Vision

- Fully separated infrastructure (network / server / user systems)
- Always-on low-power services
- Scalable homelab environment
- Real-world IT infrastructure simulation

---

## 📌 Notes

This roadmap will evolve as new hardware is acquired and new projects are implemented.
